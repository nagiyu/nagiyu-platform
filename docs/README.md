## プロジェクト全般

### 開発方針

本プロジェクトは**ドキュメント駆動開発**を採用しています。

- **理由**: AI主体の開発では、詳細なドキュメントが正確な実装を生成するための明確な指示書となる
- **アプローチ**: 要件定義 → 基本設計 → 実装の順で、各フェーズで1つのドキュメントに情報を集約
- **メリット**: 設計の一貫性を保ち、AIへの指示が明確になる
