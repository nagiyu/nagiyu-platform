# さくっとクリップ (QuickClip) 要件定義書

---

## 1. ビジネス要件

### 1.1 背景・目的

**背景**:

- 動画投稿者が長時間の動画から「見どころ」を探す作業は手間がかかり、時間を要する
- 動画の変化量・音量などの客観的指標を用いて、見どころを機械的に抽出することで、この作業を自動化・効率化したい

**目的**:

- 動画をアップロードするだけで、見どころ候補を自動抽出するサービスを提供する
- ユーザーが抽出された見どころを確認・選別・調整し、最終的に分割クリップとして書き出せるようにする

### 1.2 対象ユーザー

**プライマリーユーザー**:

- 動画投稿者（YouTuber・ニコニコ動画投稿者など）
- 長時間動画から切り抜き・ハイライト動画を作成したい

認証は不要（匿名利用）。アカウント登録・ログイン不要でサービスを利用できる。

### 1.3 ビジネスゴール

1. **見どころ抽出の自動化**: 動画の変化量・音量の大きさを分析し、見どころを自動特定する
2. **確認・選別の効率化**: 抽出された見どころをWeb画面上でプレビューしながら採否判定・時間調整できる
3. **書き出しの簡便化**: 採用した見どころを分割クリップのZIPとしてダウンロードできる
4. **将来的な高精度化**: AI文字起こしによる意味的な見どころ抽出をフェーズ2以降で実現する

---

## 2. 機能要件

### 2.1 ユースケース

#### UC-001: 動画アップロード

- **概要**: ユーザーが動画ファイルをアップロードする
- **アクター**: 動画投稿者
- **前提条件**: なし（認証不要）
- **正常フロー**:
    1. ユーザーがアップロード画面にアクセスする
    2. 動画ファイルを選択またはドラッグ&ドロップする
    3. システムがファイルを検証する（形式・サイズなど）
    4. システムがファイルをアップロードし、処理ジョブを作成する
    5. 見どころ抽出処理が開始される
- **例外フロー**:
    - ファイル形式が非対応の場合、エラーメッセージを表示してアップロードを中止する
    - ファイルサイズが上限を超える場合、エラーメッセージを表示してアップロードを中止する

#### UC-002: 見どころ確認・選別

- **概要**: 抽出された見どころをWeb画面上でプレビューし、採否を判定する
- **アクター**: 動画投稿者
- **前提条件**: UC-001 が完了し、見どころ抽出が完了している
- **正常フロー**:
    1. ユーザーが見どころ一覧画面にアクセスする
    2. システムが初回ロード時に全クリップの切り出しを自動生成する（生成中は一覧および左パネルにローディング表示）
    3. 右の一覧から見どころをクリックすると左パネルにプレビューと詳細情報が表示される
    4. 「使える」「使えない」「未確認」のいずれかを選択する（初期値は「未確認」）
    5. 必要に応じて開始・終了時刻を前後に調整する（調整確定（onBlur）でクリップが自動再生成される。確認済みのステータスは「未確認」にリセットされる）
    6. 全件の確認後、ダウンロードに進む
- **代替フロー**:
    - 一部の見どころのみ選択した状態でダウンロードできる

#### UC-003: 見どころダウンロード

- **概要**: 採用した見どころの切り出しクリップをZIPとしてダウンロードする
- **アクター**: 動画投稿者
- **前提条件**: UC-002 が完了し、1件以上の見どころが採用されており、全採用クリップの生成が完了している
- **正常フロー**:
    1. ユーザーが「ダウンロード」ボタンをクリックする
    2. システムが採用クリップをZIPにまとめてダウンロードさせる
- **例外フロー**:
    - 採用クリップに生成中のものが含まれる場合、ダウンロードボタンを無効化する

### 2.2 機能一覧

| 機能ID | 機能名 | 説明 | 優先度 |
| ------ | ------ | ---- | ------ |
| F-001 | 動画アップロード | 動画ファイルをアップロードしてジョブを作成する | 高 |
| F-002 | 見どころ自動抽出 | 変化量・音量から見どころ区間を自動抽出する（最大 20 件をスコア降順で選出後、開始時間昇順で表示）。同一時間帯が両手法で検出された場合は1件にマージして根拠を「両方」とする | 高 |
| F-003 | 見どころ一覧表示 | 抽出された見どころを開始時間昇順で一覧表示する。各行に抽出根拠（モーション・音量・両方）を表示する | 高 |
| F-004 | クリッププレビュー | 見どころをクリックして切り出し済みクリップをWeb上でプレビューする | 高 |
| F-005 | 採否チェック | 見どころに「未確認 / 使える / 使えない」の3状態でステータスを設定する。初期値は「未確認」。時間調整確定時に status が accepted/rejected の場合は unconfirmed にリセットする | 高 |
| F-006 | 時間調整 | 見どころの開始・終了時刻を前後に調整する。調整確定（onBlur）時にクリップ再生成が自動で開始される（clipStatus が GENERATING に遷移）。status が accepted/rejected の場合は unconfirmed にリセットされる | 高 |
| F-010 | クリップ自動生成 | クリップ生成は以下の場合に自動で実行される: (1) 初回表示時に全クリップが PENDING の場合、一覧取得と同時に全件の生成を開始する (2) 時間調整確定時（onBlur）に対象クリップの生成を自動で開始する。FAILED クリップのみリトライボタンを左パネルに表示し、押下で再生成する。通常フローの再生成ボタンは廃止する | 高 |
| F-007 | ZIPダウンロード | 採用した見どころの切り出し済みクリップをZIPでダウンロードする（採用クリップが全件生成済みの場合のみ実行可能） | 高 |
| F-008 | ジョブステータス確認 | 処理中のジョブのステータスを確認する | 中 |
| F-009 | AI見どころ抽出 | 文字起こしをもとにAIが見どころを抽出する | 低（Phase 2） |

### 2.3 想定画面の概要

- アップロード画面（動画ファイルの選択・アップロード）
- 処理中画面（ジョブステータスの確認）
- 見どころ一覧・確認画面（プレビュー・採否チェック・時間調整・ダウンロード）

### 2.4 クリップ生成フロー（Phase 11 UX 改善後）

- 見どころ抽出バッチはクリップ生成を行わず、ハイライトの clipStatus を `PENDING`・status を `unconfirmed` として保存する
- **初回表示時の自動生成**: 見どころ一覧取得（GET /highlights）時に PENDING のハイライトが 1 件以上存在する場合、全件の clip-regenerate Lambda を非同期 Invoke し clipStatus を `GENERATING` に更新して返却する
- **時間調整時の自動生成**: 開始・終了時刻を調整して確定（onBlur）すると PATCH API が Lambda を非同期 Invoke し clipStatus を `GENERATING` に遷移させる。status が `accepted` または `rejected` の場合は `unconfirmed` にリセットする
- API は clip-regenerate Lambda を非同期 Invoke し、clipStatus を `GENERATING` に更新して返却する
- フロントは clipStatus が `PENDING` または `GENERATING` の行をローディング表示し、`GENERATED` になるまでポーリングで状態を追跡する
- クリップの S3 パスは `outputs/{jobId}/clips/{highlightId}.mp4` として導出する（DynamoDB には保存しない）
- **FAILED 時の再試行**: clipStatus が `FAILED` のクリップに対してのみ左パネルにリトライボタンを表示する。押下で `POST /api/jobs/{jobId}/highlights/{highlightId}/regenerate` を呼び出し再生成する
- 通常フローの再生成ボタンは廃止する

## 3. 非機能要件

### 3.1 パフォーマンス要件

| 項目 | 要件 |
| ---- | ---- |
| 見どころ抽出処理時間 | 30秒以内（目安）※ Phase 2 で AI 解析追加時は別途設定 |
| 同時処理ジョブ数 | 最大3ジョブ（1ユーザーにつき1ジョブまで、同時3ユーザー想定） |
| ZIPダウンロード生成時間 | 30秒以内（目安）※ 難しければ多少の超過は許容 |
| アップロード上限 | 20 GB まで受け付けること |
| Batch 対応ファイルサイズ | 20 GB までの動画を OOM なく処理できること |
| Batch リソース選択 | ファイルサイズに応じて自動的に適切な Batch Job Definition を選択すること（< 1 GiB: small、1 GiB 以上 8 GiB 未満: large、≥ 8 GiB: xlarge） |

### 3.2 セキュリティ要件

- **データ暗号化**:
    - 転送時: HTTPS（すべての通信）
    - 保存時: S3 SSE-S3（サーバー側暗号化）
- **アクセス制御**:
    - S3バケット: 非公開（Presigned URLのみでアクセス）
    - ジョブアクセス: ジョブIDのみで制御。ジョブIDを知れば誰でも確認できるが、ジョブIDを UUID v4 等の推測困難な文字列にすることでリスクを軽減する
- **ウイルススキャン**: 実施しない（Phase 1）

### 3.3 可用性要件

AWS サービスの可用性に依存する。個別の稼働率・RTO・RPO 目標は設定しない（ベストエフォート）。

### 3.4 保守性・拡張性要件

- フェーズ2以降のAI機能追加を考慮した拡張可能なアーキテクチャとする
- 見どころ抽出アルゴリズムを差し替え・追加しやすいモジュール構成とする

### 3.5 その他の非機能要件

- **ファイル保持期間**: アップロードファイル・生成クリップともに1日（S3 Lifecycle Policy および DynamoDB TTL で自動削除）
- **対応ブラウザ**: Chrome・Edge（最新版）
- **対応デバイス**: PC（デスクトップ）のみ。スマートフォン対応・PWA 対応は行わない（codec-converter と同方針）

## 4. ドメインオブジェクト

| エンティティ | 説明 |
| ----------- | ---- |
| Job（ジョブ） | 動画のアップロードと見どころ抽出処理を管理する単位 |
| Highlight（見どころ） | 動画内の特定区間。開始時刻・終了時刻・採否ステータス（未確認 / 使える / 使えない）・クリップ生成ステータス（clipStatus）・抽出根拠（source）を持つ |
| UploadedVideo（アップロード動画） | ユーザーがアップロードした元動画ファイル |
| Clip（クリップ） | 採用された見どころから生成される分割動画ファイル |

---

## 5. スコープ外

- ❌ AI文字起こしによる見どころ抽出（Phase 1 ではスコープ外、Phase 2 以降で検討）
- ❌ 複数動画の同時アップロード（Phase 1 ではスコープ外、Phase 2 以降で検討）

---

## 6. 用語集

| 用語 | 定義 |
| ---- | ---- |
| 見どころ | 動画内で視聴価値が高いと判定された区間（開始〜終了時刻で定義） |
| 変化量 | 映像フレーム間のピクセル差分の大きさ。場面転換や激しい動きを検出するために使用 |
| クリップ | 見どころ区間を切り出した分割動画ファイル |
| QuickClip | さくっとクリップの英語サービス名 |
