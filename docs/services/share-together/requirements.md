# Share Together 要件定義

## 1. サービスの目的

### 1.1 背景

Share Together では個人 ToDo リストの名前変更・削除が可能だが、共有リストには同等の操作手段がなかった。グループメンバーが共有リストを自由に整理できるようにするため、共有リストの名前変更・削除機能を追加する。

### 1.2 対象ユーザー

- **プライマリーユーザー**: Share Together にログイン済みの、グループの承認済みメンバー

### 1.3 ビジネスゴール

- 個人リストと同等の操作性を共有リストにも提供し、ユーザー体験の一貫性を確保する

---

## 2. 機能要件

### 2.1 ユースケース

#### UC-001: 共有リスト名の変更

- **概要**: グループの承認済みメンバーが、共有リストの名前を変更する
- **アクター**: 承認済みグループメンバー
- **前提条件**: 対象グループのメンバーシップが `ACCEPTED` 状態であること
- **正常フロー**:
    1. ユーザーがサイドバーの共有リスト一覧で「編集」ボタンをクリックする
    2. 新しいリスト名を入力する
    3. リスト名が更新され、サイドバーの表示が反映される
- **代替フロー**: 名前を変更せずにキャンセルした場合は何も起きない
- **例外フロー**:
    - リスト名が空または 100 文字超の場合はバリデーションエラーを表示する
    - リストが存在しない場合はエラーメッセージを表示する
    - 非メンバーがアクセスした場合は 403 エラーを返す

#### UC-002: 共有リストの削除

- **概要**: グループの承認済みメンバーが、共有リストを削除する
- **アクター**: 承認済みグループメンバー
- **前提条件**: 対象グループのメンバーシップが `ACCEPTED` 状態であること
- **正常フロー**:
    1. ユーザーがサイドバーの共有リスト一覧で「削除」ボタンをクリックする
    2. 確認ダイアログを承認する
    3. リストおよびそのリストに属する ToDo が削除される
    4. サイドバーの表示が反映され、別のリストに遷移する（存在する場合）
- **代替フロー**: 確認ダイアログをキャンセルした場合は何も起きない
- **例外フロー**:
    - リストが存在しない場合はエラーメッセージを表示する
    - 非メンバーがアクセスした場合は 403 エラーを返す

### 2.2 機能一覧

| 機能 ID | 機能名 | 説明 | 優先度 |
| ------- | ------ | ---- | ------ |
| F-001 | 共有リスト名の変更 | 承認済みメンバーが共有リストの名前を変更できる | 高 |
| F-002 | 共有リストの削除 | 承認済みメンバーが共有リストを削除できる（配下の ToDo も削除） | 高 |

---

## 3. ドメインオブジェクト

| エンティティ | 説明 |
| ----------- | ---- |
| GroupList | グループに紐づく共有 ToDo リスト |
| GroupMembership | ユーザーとグループの紐付け（ACCEPTED のメンバーのみ操作可能） |

---

## 4. スコープ外

- グループオーナーのみが操作可能にする権限管理の細分化（全メンバーが操作可能とする）
- 削除した共有リストの復元
- 個人リスト操作ロジックの変更

---

## 5. 受け入れ基準

- 共有スコープのサイドバーで、各共有リストに編集・削除アイコンボタンが表示される
- 編集ボタンクリック後に新しい名前を入力し確定すると、サイドバーの表示名が更新される
- 削除ボタンクリック後に確認を承認すると、リストがサイドバーから消え、別リストに遷移する
- 操作成功・失敗はスナックバーで通知される
- 非メンバーが直接 API を呼び出しても 403 が返ることを確認できる
