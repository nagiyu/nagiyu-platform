# 設計判断記録（ADR: Architecture Decision Records）

各判断は以下の形式で記録されます：
- 判断日
- 判断内容
- 背景・理由
- 代替案
- 結果・影響

## 記録例（参考）

### ADR-000: ドキュメント駆動開発の採用

**日付**: 2025-12-14

**ステータス**: 承認済み

#### 背景・課題
AI主体での開発を効率的に進めるため、開発手法を検討する必要があった。

#### 判断内容
ウォーターフォール型のドキュメント駆動開発を採用する。

#### 代替案

**案1: アジャイル開発**
- メリット: 柔軟な要件変更、短期リリース
- デメリット: AIへの指示が断片的になりがち

**案2: コードファースト開発**
- メリット: 早期実装開始
- デメリット: AI主体の開発で設計ブレが発生しやすい

#### 選定理由
- AIは詳細なドキュメントから正確な実装を生成できる
- フェーズごとのドキュメントがAIへの明確な指示書になる
- 設計の一貫性を保ちやすい

#### 結果・影響
- 各フェーズで1つのドキュメントに情報を集約
- 要件定義→基本設計→詳細設計→実装の順で進める
