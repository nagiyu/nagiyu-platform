# Portal 要件定義書

---

## 1. ビジネス要件

### 1.1 背景・目的

nagiyu.com（ルートドメイン）に Google AdSense 承認を取得するために、
クローラブルなテキストコンテンツを豊富に持つポータルサービスを新設する。

現在ルートドメインで稼働している Tools（7 種のブラウザ完結型ユーティリティ）は
`tools.nagiyu.com` に移行する（インフラは既存のものを使用）。

Portal は「ブログ（定期更新必須）」ではなく **サービスドキュメント（マニュアル）＋技術紹介記事** を主体とする。
更新はサービスの機能追加・変更に合わせて行うため、定期的な記事投稿は不要。

### 1.2 対象ユーザー

- **プライマリ**: nagiyu の各サービスを利用する開発者・一般ユーザー
- **セカンダリ**: Google AdSense 審査担当者（コンテンツの有用性・E-A-T を評価）

### 1.3 ビジネスゴール

1. **Google AdSense 承認取得**（nagiyu.com ドメインで承認後、各サブドメインへ展開）
2. **SEO トラフィック獲得**（サービスドキュメントがロングテールキーワードで流入を生む）
3. **ユーザーへのサービス紹介・導線整備**（各サービスへの入口を一元化）

## 2. 機能要件

### 2.1 ユースケース

#### UC-001: サービス一覧の閲覧

- **概要**: ユーザーが nagiyu.com を訪問し、提供されている全サービスを一覧で確認する
- **アクター**: 一般ユーザー
- **前提条件**: なし（認証不要）
- **正常フロー**:
    1. ユーザーが nagiyu.com にアクセスする
    2. トップページに 8 サービスのカード一覧が表示される
    3. ユーザーが興味のあるサービスのカードをクリックする
    4. サービス概要ページへ遷移する
- **代替フロー**: なし

#### UC-002: サービスドキュメントの閲覧

- **概要**: ユーザーが特定のサービスの使い方を知るためにドキュメントを参照する
- **アクター**: 一般ユーザー
- **前提条件**: なし（認証不要）
- **正常フロー**:
    1. ユーザーがサービス概要ページ（`/services/[slug]`）にアクセスする
    2. サービスの概要・特徴が表示される
    3. ユーザーが「使い方ガイド」リンクをクリックする
    4. 手順やスクリーンショットを含む詳細ガイドが表示される
- **代替フロー**: FAQ ページへの遷移

#### UC-003: 技術記事の閲覧

- **概要**: 開発者・技術者がサービスの実装背景や技術解説を読む
- **アクター**: 開発者・技術者
- **前提条件**: なし（認証不要）
- **正常フロー**:
    1. ユーザーが技術記事一覧（`/tech`）にアクセスする
    2. 記事タイトル・概要・タグ一覧が表示される
    3. ユーザーが記事を選択する
    4. Markdown から変換された記事本文が表示される

## 3. 非機能要件

### 3.1 パフォーマンス要件

| 項目 | 要件 |
| ---- | ---- |
| Lighthouse Performance スコア | 90 以上（AdSense 審査・SEO 要件） |
| Lighthouse Accessibility スコア | 90 以上 |
| Core Web Vitals | LCP < 2.5s、FID < 100ms、CLS < 0.1 |

### 3.2 セキュリティ要件

- 認証不要（全ページ公開）
- 外部スクリプト（AdSense / Analytics）は本番環境のみ読み込む
- セキュリティヘッダーを CloudFront レベルで付与（既存インフラの設定を継承）

### 3.3 可用性要件

| 項目 | 要件 |
| ---- | ---- |
| 稼働率 | 99.9%（既存インフラ共通） |
| RTO | 30 分 |
| RPO | N/A（ステートレス・静的コンテンツ） |

### 3.4 保守性・拡張性要件

- コンテンツ追加・更新は Markdown ファイルの編集のみで完結すること
- 新サービスの追加時は `content/services/` 配下にディレクトリを追加するだけでよい構成
- 既存サービスパターンに準拠（Next.js 15 + MUI 6 + TypeScript strict）

### 3.5 SEO・クローラビリティ要件

- SSG（静的サイト生成）による完全クローラブルなページ生成
- 各ページに適切な `<title>` / `<meta description>` / OGP タグを設定
- サイトマップ（`/sitemap.xml`）の自動生成
- robots.txt の設置

## 4. ドメインオブジェクト

| エンティティ | 説明 |
| ----------- | ---- |
| Service | nagiyu が提供するサービス（Tools、Quick-Clip など 8 種） |
| ServiceDocument | サービスに紐づく Markdown ドキュメント（概要・ガイド・FAQ の 3 種） |
| Article | 技術紹介記事（Markdown ファイル、タグ・公開日付き） |

---

## 5. スコープ外

- ❌ コンテンツ管理システム（CMS）の導入
- ❌ ユーザー認証・会員機能
- ❌ コメント・いいね機能
- ❌ 各サービスの機能実装（Portal は紹介・ドキュメントのみ）
- ❌ 動的なコンテンツ生成（全ページ静的）

## 6. 用語集

| 用語 | 定義 |
| ---- | ---- |
| Portal | nagiyu.com のルートドメインで稼働するサービス紹介・ドキュメントサイト |
| SSG | Static Site Generation。ビルド時に静的 HTML を生成する方式 |
| E-A-T | Expertise（専門性）・Authoritativeness（権威性）・Trustworthiness（信頼性）。Google の品質評価基準 |
| AdSense | Google の広告配信プログラム。サイトオーナーが承認を受けることで広告収益を得られる |
