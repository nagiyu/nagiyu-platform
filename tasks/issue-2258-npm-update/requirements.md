# NPM パッケージ更新 (2026年第11週) 要件定義書

---

## 1. ビジネス要件

### 1.1 背景・目的

2026-03-16 に自動生成された週次 npm 管理レポート（Issue #2258）に基づき、
プラットフォーム全体の依存パッケージを最新安定版へ更新する。
脆弱性・バージョン不整合は検出されなかったため、本作業は Priority 3（改善推奨）に分類される。
パッケージを定期的に更新することで、セキュリティリスクの抑制と将来の大規模アップグレードコストを低減する。

### 1.2 対象ユーザー

- 開発者（依存バージョンを最新に保ちたい）
- プラットフォーム運用担当者（セキュリティ・安定性の維持）

### 1.3 ビジネスゴール

- 全ワークスペースのパッケージを最新安定版へ更新する
- CI（lint / build / test）がすべて通過した状態を維持する

---

## 2. 機能要件

### 2.1 ユースケース

#### UC-001: パッケージの一括更新

- **概要**: ルート・各ワークスペースの `package.json` バージョンを最新安定版へ引き上げ、`npm install` で `package-lock.json` を再生成する
- **アクター**: 開発者
- **前提条件**: `npm outdated` でバージョンアップ可能なパッケージが存在している
- **正常フロー**:
    1. 更新対象パッケージを優先度別にグループ化する
    2. 各グループに対して `npm update` または `npm install {pkg}@{version}` を実行する
    3. `npm install` でロックファイルを更新する
    4. lint / build / test を実行して動作を確認する
    5. 問題がなければコミットする
- **代替フロー**: メジャーバージョン変更（eslint v10、@types/node v25 等）は breaking changes を確認してから個別対応する
- **例外フロー**: CI 失敗が発生した場合は該当パッケージの更新を切り戻し、原因を調査する

## 3. 非機能要件

### 3.1 パフォーマンス要件

| 項目 | 要件 |
| ---- | ---- |
| CI 実行時間 | 更新前後で大幅な増加がないこと |

### 3.2 セキュリティ要件

- `npm audit` の結果が Critical / High ゼロを維持すること
- メジャーバージョン変更により新たな脆弱性が混入しないこと

### 3.3 可用性要件

| 項目 | 要件 |
| ---- | ---- |
| CI 合格率 | 更新後に全 CI（lint / build / test / E2E）が合格すること |

### 3.4 保守性・拡張性要件

- `package-lock.json` が更新されリポジトリに反映されること
- メジャーバージョン変更は個別の Issue／PR として切り出すことを許容する

---

## 4. ドメインオブジェクト

| エンティティ | 説明 |
| ------------ | ---- |
| パッケージ | npm レジストリ上の依存ライブラリ（name + version） |
| ワークスペース | モノレポ内の個別パッケージ（infra, libs, services 配下） |
| ロックファイル | `package-lock.json`。インストールバージョンを固定する |

---

## 5. スコープ外

- ❌ `next-auth` (5.0.0-beta.30) の安定版への移行（別 Issue 対応）
- ❌ `@auth/core` のバージョン変更（next-auth との整合性が必要なため別途検討）
- ❌ `eslint` v10 / `@eslint/js` v10 へのアップグレード（メジャー変更のため調査後に別 Issue で対応）
- ❌ `@types/node` v25 へのアップグレード（メジャー変更のため調査後に別 Issue で対応）

## 6. 用語集

| 用語 | 定義 |
| ---- | ---- |
| npm outdated | 現在インストール済みのバージョンと最新バージョンを比較するコマンド |
| semver | Semantic Versioning（major.minor.patch）形式のバージョン管理規則 |
| breaking change | メジャーバージョンアップ時に後方互換性を失う変更 |
