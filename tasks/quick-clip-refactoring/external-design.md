## 変更対象画面

既存の画面設計（`docs/services/quick-clip/external-design.md`）に対して以下の変更を加える。

---

## SCR-001: アップロード画面（変更）

### 変更内容

**追加する UI 要素**

| 要素 | 種別 | 表示条件 |
|-----|------|---------|
| 制約通知エリア | Alert（info） | 常時表示 |
| アップロード進捗バー | LinearProgress | `isSubmitting === true` |
| 進捗テキスト | テキスト | `isSubmitting === true` |

**制約通知（常時表示）**

ファイル選択エリアの上部に MUI `Alert` (severity="info") で以下を表示する:

```
ℹ  アップロード中はタブを閉じないでください。
   データは 24 時間で自動削除されます。
```

**アップロード進捗（アップロード中のみ表示）**

アップロードボタンの上に `LinearProgress` と進捗テキストを表示する:

```
[██████████░░░░░░░░░░] 54%
アップロード中... 54%
```

アップロードボタンのラベルを動的に変更する:

| 状態 | ラベル |
|------|--------|
| 待機中 | アップロードして処理開始 |
| アップロード中（0〜99%） | アップロード中... 54% |
| アップロード完了・処理開始中（100%） | 処理開始中... |
| アップロード未選択 | アップロードして処理開始（disabled） |

**既存の表示条件変更**

`docs/services/quick-clip/external-design.md` SCR-001 の以下の記述を変更する:
- 旧: 「ローディング: アップロード中はローディングスピナーを表示（アップロードの進捗取得は行わない）」
- 新: 「ローディング: アップロード中はプログレスバーで進捗（0〜100%）を表示する」

## SCR-002: 処理中画面（変更）

### 変更内容

**追加する UI 要素**

| 要素 | 種別 | 表示条件 |
|-----|------|---------|
| フロー進捗ステッパー | MUI Stepper（水平） | 常時表示 |
| タブ閉じOK通知 | テキスト | PENDING / PROCESSING 時 |
| 有効期限表示 | テキスト | 常時表示 |

**フロー進捗ステッパー**

ページ上部に MUI `Stepper` を水平配置する。3 ステップ:

| ステップ | ラベル | サブラベル | 条件 |
|---------|--------|-----------|------|
| 1 | アップロード | なし | 常に completed |
| 2 | 解析 | batchStage のラベル | PENDING / PROCESSING 時に active |
| 3 | 切り出し | なし | COMPLETED 時に active |

`batchStage` のサブラベルマッピング:

| batchStage | 表示テキスト |
|-----------|------------|
| `downloading` | ダウンロード中 |
| `analyzing` | 解析中 |
| `aggregating` | 集計中 |
| 未設定 | （表示なし） |

**通知テキスト（PENDING / PROCESSING 時）**

```
タブを閉じても処理は続きます。URLを控えておくと後で確認できます。
```

**有効期限表示**

```
データの有効期限: 2024/04/22 15:30
```

`job.expiresAt`（Unix 秒）を `toLocaleString('ja-JP')` で変換して表示する。

**既存の表示条件変更**

`docs/services/quick-clip/external-design.md` SCR-002 の以下の記述を変更する:
- 旧: 「ローディングスピナー … 10秒毎に自動ポーリングでステータスを更新」
- 新: 「ローディングスピナー … 5秒毎に自動ポーリングでステータスを更新」

## SCR-003: 見どころ確認画面（変更）

### 変更内容

**追加する UI 要素**

| 要素 | 種別 | 表示条件 |
|-----|------|---------|
| クリップ生成進捗 | テキスト + CircularProgress | `hasPendingOrGenerating === true` |
| 有効期限表示 | テキスト | 常時表示 |

**クリップ生成進捗**

テーブル上部（ColumnVisibilityButton の横）に表示:

```
⟳  3 / 20 件生成済
```

`hasPendingOrGenerating` が false になったら非表示にする。

**有効期限表示**

ページ上部（テーブルヘッダー付近）に表示:

```
データの有効期限: 2024/04/22 15:30
```

`job.expiresAt` から算出。SCR-003 では `highlights` レスポンスから `job.expiresAt` を取得できるよう、ハイライト一覧 API がジョブの `expiresAt` を含むレスポンスを返すか、別途ジョブ情報をフェッチする。

> **実装ノート**: 現在の SCR-003 はジョブ情報を持っていないため、`expiresAt` の取得方法を design.md で検討すること。最もシンプルな方法は、URL パラメータの `jobId` を使って `GET /api/jobs/[jobId]` を初回マウント時に 1 回呼ぶ方法。

---

## 設計上の注意点

- MUI の `Stepper`・`Step`・`StepLabel` は既存の MUI 導入済み環境でそのまま使用できる
- `LinearProgress` も MUI に含まれており、追加インストール不要
- `expiresAt` は Unix 秒（number 型）。`new Date(expiresAt * 1000).toLocaleString('ja-JP')` で日本語フォーマットの日時文字列を得る
