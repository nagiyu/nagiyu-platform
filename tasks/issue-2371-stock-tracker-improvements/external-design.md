# Stock Tracker 改善 - 外部設計（Issue #2371）

## 1. 変更対象画面

本改修で変更・追加が発生する画面は以下の通り。

| 画面 ID | 画面名 | 変更種別 |
|--------|--------|---------|
| SCR-001 | トップ画面（チャート表示） | 変更（サマリーパネル・保有株式パネル・アラートパネル） |
| SCR-004 | サマリー一覧画面 | 変更（詳細ダイアログのモバイル対応バグ修正） |

---

## 2. SCR-001: トップ画面 変更仕様

### 2.1 日次サマリーパネル（変更）

**変更前**: OHLCV 等の価格データを表示

**変更後**: 投資判断に有用な情報を表示

**主要 UI 要素（変更後）**

| 要素 | 種別 | 説明 |
|-----|------|------|
| 投資判断 | テキスト／バッジ | 買い・売り・中立などの判定結果を色付きで表示する |
| 買いシグナル | テキスト／バッジ | 買いシグナルの件数を表示する |
| 売りシグナル | テキスト／バッジ | 売りシグナルの件数を表示する |
| AI 判定 | テキスト | AI 解析による総合判断の要約を表示する |
| 更新時間 | テキスト | サマリーの最終更新日時を表示する |
| サポートレベル | テキスト | 価格のサポート水準（データがある場合のみ表示） |
| レジスタンスレベル | テキスト | 価格のレジスタンス水準（データがある場合のみ表示） |
| 詳細ボタン | ボタン | クリックでサマリー詳細ダイアログを開く |

**ユーザーインタラクション（追加）**

| 操作 | 結果 |
|------|------|
| 詳細ボタンをクリック | 選択ティッカーのサマリー詳細ダイアログが表示される（SCR-004 のダイアログを流用） |

**表示条件・状態（変更後）**

- サマリーデータなし: 「サマリーデータがありません」を表示する
- AI 判定未生成: AI 判定エリアに「AI 解析はまだ生成されていません」を表示する
- サポート・レジスタンスレベルは、データが存在する場合のみ表示行を出す

---

### 2.2 保有株式パネル（変更）

**変更前**: 保有情報の閲覧のみ

**変更後**: 保有情報の閲覧 + 作成・修正・削除（CRUD）

**追加 UI 要素**

| 要素 | 種別 | 説明 |
|-----|------|------|
| 保有追加ボタン | ボタン | クリックで保有株式作成ダイアログを開く |
| 編集ボタン | ボタン（行単位） | クリックで対象保有株式の編集ダイアログを開く |
| 削除ボタン | ボタン（行単位） | クリックで確認後に保有株式を削除する |

**ユーザーインタラクション（追加）**

| 操作 | 結果 |
|------|------|
| 保有追加ボタンをクリック | 保有株式作成ダイアログが開く（取引所・ティッカーは現在選択中の値をデフォルトとして設定） |
| 編集ボタンをクリック | 保有株式編集ダイアログが開く |
| 削除ボタンをクリック | 確認ダイアログが表示され、確認後に保有株式が削除される |
| ダイアログで保存 | 保有株式パネルの表示が更新される |

**流用コンポーネント**: SCR-002（保有株式管理画面）の作成・編集・削除ダイアログ

---

**変更前**: アラート情報の閲覧のみ

**変更後**: アラート情報の閲覧 + 作成・修正・削除（CRUD）

**追加 UI 要素**

| 要素 | 種別 | 説明 |

## 3. SCR-004: サマリー一覧画面 バグ修正

### 3.1 詳細ダイアログ モバイル横幅オーバー修正

**問題**: モバイル幅でサマリー詳細ダイアログを表示した際に、ダイアログが画面横幅を超えて表示され横スクロールが必要になる。

**修正方針**

| 対象 | 修正内容 |
|------|---------|
| ダイアログ全体 | `maxWidth` を `100vw` 以内に収め `overflow-x: hidden` を設定する |
| ダイアログ内コンテンツ | 横幅を `100%` 基準にして親を超えないよう制御する |
| テーブル等の横長要素 | `overflow-x: auto` でスクロール可能にし、親幅を超えないようにする |
| 長文テキスト | `word-break: break-word` または `overflow-wrap: break-word` で折り返す |

**検証環境**:
- モバイル幅（375px 程度）でのダイアログ表示
- iPhone SE、Galaxy S 系等の狭い画面でのレイアウト確認

## 4. レスポンシブ方針（変更なし）

既存方針を維持する。

- モバイル: 縦スクロールを基本とし、追加されたボタン類はパネル内に適切に配置する
- デスクトップ: 横方向レイアウトを維持しつつ、CRUD ボタンを各パネルに収める
