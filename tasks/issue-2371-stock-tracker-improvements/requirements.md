# Stock Tracker 改善 - 要件定義（Issue #2371）

## 1. 変更スコープの概要

本ドキュメントは Issue #2371「Stock Tracker の改善」の変更スコープを定義する。
既存ユースケースの変更と新規機能追加の両方を含む。

---

## 2. ユースケース変更・追加

### UC-010 改訂: 一時アラート失効処理（無効化 → 削除）

**概要**: 期限切れの一時アラートを自動的に**削除**する（現行の「無効化」から変更）。

**変更内容**:
- 現行: 期限切れアラートの `Enabled` フラグを `false` にする
- 変更後: 期限切れアラートを DynamoDB から**削除**する

**前提条件（変更なし）**:
- DynamoDB に一時アラートが存在する
- バッチが毎時起動する

**正常フロー（変更点のみ）**:
1. バッチが毎時起動し、通知頻度ごとに全件の一時アラートを取得する（ページネーション対応）
2. 各アラートを評価し、期限切れ・取引時間外かを確認する
3. 条件を満たすアラートを**削除**する

**代替フロー・例外フロー（変更なし）**:
- 対象アラートがなければスキップして正常終了
- マッピング不能な不正データは識別情報をワーニングログ出力してスキップし後続処理を継続
- 個別アラートの削除に失敗した場合はログ出力してスキップし後続処理を継続

---

### UC-009 改訂: チャート画面統合表示（サマリーパネル表示内容変更）

**概要**: トップ画面の日次サマリーパネルに表示する項目を投資判断に有用な情報に絞り込む。

**変更内容**:
- 現行: OHLCV 等の価格データをサマリーパネルに表示
- 変更後: 下記の情報を表示する

| 表示項目 | 説明 |
|---------|------|
| 投資判断 | 買い・売り・中立などの判定結果 |
| 買いシグナル | 買いシグナルの有無・件数 |
| 売りシグナル | 売りシグナルの有無・件数 |
| AI 判定 | AI 解析による総合判断 |
| 更新時間 | サマリーが最後に更新された日時 |
| サポートレベル | （任意）価格のサポート水準 |
| レジスタンスレベル | （任意）価格のレジスタンス水準 |

**詳細ダイアログ（新規追加）**:
- サマリーパネルに「詳細」ボタンを追加する
- ボタン押下で該当ティッカーのサマリー詳細ダイアログを表示する
- ダイアログの内容は既存の SCR-004（サマリー一覧画面）の詳細ダイアログを流用する

---

### UC-003 拡張: トップ画面からの保有株式 CRUD

**概要**: トップ画面の保有株式パネルから、保有株式の作成・修正・削除を行えるようにする。

**追加要件**:
- 保有株式パネルに「追加」ボタンを表示する（保有なし・あり問わず）
- 保有株式が存在する場合、各行に「編集」「削除」ボタンを表示する
- 既存 SCR-002（保有株式管理画面）のダイアログコンポーネントを流用する

---

### UC-002 拡張: トップ画面からのアラート CRUD

**概要**: トップ画面のアラートパネルから、アラートの作成・修正・削除を行えるようにする。

**追加要件**:
- アラートパネルに「追加」ボタンを表示する
- アラートが存在する場合、各行に「編集」「削除」ボタンを表示する
- 既存 SCR-003（アラート一覧画面）のダイアログコンポーネントを流用する

---

### バグ修正: サマリー詳細ダイアログのモバイル横幅オーバー

**概要**: モバイルでサマリー詳細ダイアログを表示した際に横幅を超えて表示され、横スクロールが発生するバグを修正する。

**再現条件**:
- モバイル幅でサマリー詳細ダイアログを開く

**修正方針**:
- ダイアログおよびダイアログ内コンテンツの最大幅・オーバーフロー設定を見直す
- モバイル幅でダイアログが画面幅内に収まるよう修正する
- テーブル等の横長コンテンツは折り返し・スクロール可能にする

---

### UC-006 改訂: AI 解析の Web 検索必須化

**概要**: サマリーの AI 解析において、関連市場・セクター動向の調査に Web 検索を必ず実施するよう変更する。

**変更内容**:
- 現行: `tool_choice` 未設定のため Web 検索が任意実行
- 変更後: OpenAI Responses API の `tool_choice` を `required` に設定し Web 検索を強制
- プロンプトの `relatedMarketTrend` フィールド説明を「必ずWeb検索を利用」に変更
- 可能であれば、決算等の直近で動向が変わる可能性のあるニュースも根拠として含める

---

## 3. 機能要件

| ID | 要件 |
|----|------|
| FR-2371-01 | 期限切れ一時アラートは `Enabled` フラグの無効化ではなく削除を行う |
| FR-2371-02 | トップ画面の日次サマリーパネルに投資判断・買いシグナル・売りシグナル・AI判定・更新時間を表示する |
| FR-2371-03 | トップ画面の日次サマリーパネルにサポートレベル・レジスタンスレベルを表示する（可能な場合） |
| FR-2371-04 | トップ画面のサマリーパネルに詳細ダイアログ表示ボタンを設け、SCR-004 の詳細ダイアログを流用して表示する |
| FR-2371-05 | トップ画面の保有株式パネルから保有株式の作成・修正・削除を行えるようにする |
| FR-2371-06 | トップ画面のアラートパネルからアラートの作成・修正・削除を行えるようにする |
| FR-2371-07 | モバイル幅でサマリー詳細ダイアログが横幅を超えないよう修正する |
| FR-2371-08 | AI 解析時に Web 検索を常に実施する（`tool_choice: required`） |
| FR-2371-09 | AI 解析のプロンプトに決算等の直近ニュースを根拠として含める指示を追加する（可能な場合） |

## 4. 非機能要件

| ID | 要件 |
|----|------|
| NFR-2371-01 | 一時アラート失効バッチの削除処理は、個別失敗時にスキップ継続する冪等設計を維持する |
| NFR-2371-02 | トップ画面への CRUD 追加後もモバイルでの縦スクロール基本レイアウトを維持する |
| NFR-2371-03 | ダイアログコンポーネントの流用は既存コンポーネントの再利用性を損なわない形で行う |
| NFR-2371-04 | Web 検索強制により AI 解析の応答時間が増加する可能性を許容する |
