# クリップ一覧の列オプショナル化 - 外部設計

---

## 1. 画面設計

### 1.1 変更対象画面

本タスクで変更する画面は SCR-003（見どころ確認画面）の右パネル（マスター一覧）のみ。
他画面への変更はない。

### 1.2 SCR-003 右パネル（マスター一覧）の変更

#### 変更前の列構成

```
┌──────┬──────────────────┬──────────┐
│ No.  │ 開始〜終了(秒)    │ 採否     │
├──────┼──────────────────┼──────────┤
│  #1  │ 0:10 〜 0:20     │ [未確認] │
│  #2  │ 0:30 〜 0:40     │ [使える] │
└──────┴──────────────────┴──────────┘
```

#### 変更後の列構成（抽出根拠列 OFF 時・デフォルト）

変更前と同一の見た目を維持する。テーブル右上に「列設定」ボタンを追加する。

```
┌──────┬──────────────────┬──────────┐  [⚙ 列設定]
│ No.  │ 開始〜終了(秒)    │ 採否     │
├──────┼──────────────────┼──────────┤
│  #1  │ 0:10 〜 0:20     │ [未確認] │
│  #2  │ 0:30 〜 0:40     │ [使える] │
└──────┴──────────────────┴──────────┘
```

#### 変更後の列構成（抽出根拠列 ON 時）

```
┌──────┬──────────────────┬──────────┬──────────┐  [⚙ 列設定]
│ No.  │ 開始〜終了(秒)    │ 採否     │ 抽出根拠 │
├──────┼──────────────────┼──────────┼──────────┤
│  #1  │ 0:10 〜 0:20     │ [未確認] │ モーション│
│  #2  │ 0:30 〜 0:40     │ [使える] │ 音量     │
│  #3  │ 0:45 〜 0:55     │ [未確認] │ 両方     │
└──────┴──────────────────┴──────────┴──────────┘
```

### 1.3 列設定 UI の設計

#### 列設定ボタン

- テーブルヘッダー行の右上（テーブル外側）に配置する
- アイコンボタン（設定アイコン）またはテキストボタン「列設定」とする

#### 列設定ドロップダウン（ポップオーバー）

- 列設定ボタンをクリックすると、ボタン直下にポップオーバーが展開する
- オプション列のみチェックボックス形式で表示する（固定列は表示しない）
- チェック状態の変更は即座にテーブルへ反映する

```
┌─────────────────┐
│ ✓ 抽出根拠      │
│ （将来の追加列） │
└─────────────────┘
```

**UI 要素一覧**

| 要素             | 種別           | 説明                                               |
| ---------------- | -------------- | -------------------------------------------------- |
| 列設定ボタン     | アイコンボタン | クリックでポップオーバーを開閉する                 |
| ポップオーバー   | ポップオーバー | オプション列のチェックボックス一覧を表示する       |
| 抽出根拠チェック | チェックボックス | ON でテーブルに「抽出根拠」列を追加する（デフォルト OFF） |

**ユーザーインタラクション**

| 操作                         | 結果                                             |
| ---------------------------- | ------------------------------------------------ |
| 列設定ボタンをクリック       | ポップオーバーが開く                             |
| ポップオーバー外をクリック   | ポップオーバーが閉じる                           |
| 「抽出根拠」をチェック       | テーブルに抽出根拠列が即座に追加される           |
| 「抽出根拠」のチェックを外す | テーブルから抽出根拠列が即座に非表示になる       |

### 1.4 抽出根拠列の表示内容

| source 値 | 表示文字列 |
| --------- | ---------- |
| `motion`  | モーション |
| `volume`  | 音量       |
| `both`    | 両方       |

> `source` の表示ラベルは既存の `HIGHLIGHT_SOURCE_LABELS` 定数を流用する。

## 2. 概念データモデル

本タスクによるデータモデルの変更はない。
`Highlight.source` 属性（既存）を列として表示するのみ。

## 3. 設計上の決定事項（ADR）

### ADR-101: オプション列のデフォルトを OFF にする

**背景・問題**

テーブルに「抽出根拠」列を常時追加すると、表が横に広がり、主要情報（No.・開始〜終了・採否）の視認性が低下する。

**決定**

「抽出根拠」列はデフォルト OFF とし、ユーザーが明示的に有効化した場合のみ表示する。

**根拠・トレードオフ**

- メリット: デフォルト状態でのテーブル幅を現状維持でき、多数のクリップがある場合も一覧性を保てる
- デメリット: 初回利用時にユーザーが機能に気づきにくい可能性がある
- 対策: 列設定ボタンをテーブル右上に配置し、視覚的に発見しやすくする
