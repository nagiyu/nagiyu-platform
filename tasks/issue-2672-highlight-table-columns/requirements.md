# クリップ一覧の列オプショナル化 - 要件定義

---

## 1. ビジネス要件

### 1.1 背景・目的

**背景**:

- ハイライト確認画面（SCR-003）の右パネル（マスター一覧）テーブルに「抽出根拠（モーション / 音量 / 両方）」列が存在せず、
  ユーザーがクリップ選定の根拠を一覧上で確認できない問題がある
- 一方でテーブルに列を追加し続けると表が肥大化し、かえって視認性が低下する

**目的**:

- 「抽出根拠」列をオプション列として追加し、ユーザーが表示 / 非表示を自由に切り替えられるようにする
- デフォルトではテーブルをコンパクトに保ちつつ、必要なときに追加情報を表示できる柔軟な UI を提供する

### 1.2 対象ユーザー

既存の対象ユーザーと同一（docs/services/quick-clip/requirements.md 1.2 節参照）。

### 1.3 ビジネスゴール

- テーブルの視認性を保ちながら、クリップ選定に必要な情報へのアクセスを改善する

---

## 2. 機能要件

### 2.1 ユースケース

#### UC-101: 列表示の切り替え

- **概要**: ユーザーがクリップ一覧テーブルの列表示を ON / OFF で切り替える
- **アクター**: 動画投稿者
- **前提条件**: ハイライト確認画面（SCR-003）を表示している
- **正常フロー**:
    1. ユーザーがテーブル上の「列設定」ボタンをクリックする
    2. 表示 / 非表示を切り替え可能な列の一覧がドロップダウン（またはポップオーバー）で表示される
    3. ユーザーが任意の列をチェック / アンチェックする
    4. テーブルが即座に更新され、チェックされた列のみ表示される
    5. 設定が localStorage に保存され、次回アクセス時も維持される
- **代替フロー**:
    - localStorage が利用不可な場合: 設定はセッション内のみ保持し、警告なしにフォールバックする

### 2.2 機能一覧

本タスクで追加・変更する機能のみ記載する。

| 機能ID  | 機能名                   | 説明                                                                                              | 優先度 |
| ------- | ------------------------ | ------------------------------------------------------------------------------------------------- | ------ |
| F-003   | 見どころ一覧表示（更新） | 各行に抽出根拠を表示する列を追加する。列はオプションとしてデフォルト OFF で追加する               | 高     |
| F-014   | 列表示設定               | テーブルの各列の表示 / 非表示をユーザーが切り替えられる UI を提供する                             | 高     |
| F-015   | 列設定の永続化           | 列表示設定を localStorage に保存し、次回アクセス時に復元する                                      | 中     |

### 2.3 各列のデフォルト表示状態

| 列名           | デフォルト | 変更可否 |
| -------------- | ---------- | -------- |
| No.            | ON         | 不可     |
| 開始〜終了(秒) | ON         | 不可     |
| 採否           | ON         | 不可     |
| 抽出根拠       | OFF        | 可       |

> No.・開始〜終了・採否の3列は常時表示とし、非表示に切り替えることはできない。

---

## 3. 非機能要件

### 3.1 パフォーマンス要件

| 項目             | 要件                                         |
| ---------------- | -------------------------------------------- |
| 列切り替えの応答 | ユーザー操作から 100ms 以内にテーブルを更新する |

### 3.2 セキュリティ要件

- localStorage への保存はユーザー設定（列表示状態）のみとし、機密データは含めない

### 3.3 保守性・拡張性要件

- 将来的に追加されるオプション列も同じ仕組みで対応できること
- 列の定義をデータとして管理し、ハードコードを避けること

---

## 4. ドメインオブジェクト

本タスクで新規に登場するドメインオブジェクトはない。
既存の `Highlight.source` 属性（`motion` / `volume` / `both`）を列として表示する。

---

## 5. スコープ外

- ❌ 列の並び替え（ドラッグ&ドロップ等）
- ❌ 列幅のリサイズ
- ❌ No.・開始〜終了・採否 列の非表示化
- ❌ ユーザーアカウントへの設定同期（localStorage のみ）

---

## 6. 受け入れ条件

- [ ] ハイライト確認画面（SCR-003）のテーブルに「抽出根拠」列が追加されている
- [ ] 「抽出根拠」列はデフォルトで非表示である
- [ ] 列設定 UI から「抽出根拠」列の表示 / 非表示を切り替えられる
- [ ] 切り替えはテーブルに即座に反映される
- [ ] 設定が localStorage に保存され、ページ再読み込み後も維持される
- [ ] 既存の No.・開始〜終了・採否 列は常時表示であり、列設定 UI に現れない
- [ ] Lint・型チェックが通過している
