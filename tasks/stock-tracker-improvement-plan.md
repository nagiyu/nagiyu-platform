# Stock Tracker 改善計画書

## 概要

Stock Tracker サービスの機能拡充・品質向上を目的とした改善計画。
バッチ処理の安定性向上、チャート機能の強化、サマリー機能の拡張、ウォッチリスト機能の刷新などを含む。

- 親Issue: #1778（Stock Tracker の改善）
- タスクタイプ: サービスタスク（stock-tracker）
- バージョン目標: 2.0.0（#1786）

## 要件

### 機能要件

- FR1: バッチ更新処理を取引所ごと・ティッカーごとにループし、登録すべきサマリーの有無を判定して差分更新できること（#1780）
- FR2: AI 解析と静的解析（OHLC・パターン）を分離して判定・更新できること（#1780）
- FR3: 各テクニカルパターンを個別に判定し、新規パターン追加時でも再解析が可能なこと（#1780）
- FR4: 取引所の営業時間設定でサマータイムを考慮できること（#1779）
- FR5: チャートをコンポーネント化し、複数画面へ展開できること（#1781）
- FR6: チャート表示内容を改善すること（#1782）
- FR7: サマリーにテクニカルパターンを追加表示できること（#1783）
- FR8: サマリーの表示内容変更とアラート設定機能を追加すること（#1784）
- FR9: ウォッチリスト機能をモック/ハードコーディングから脱却し、実データに基づいて動作すること（#1785）

### 非機能要件

- NFR1: TypeScript strict mode 必須
- NFR2: テストカバレッジ 80% 以上（Jest `coverageThreshold` で自動検証）
- NFR3: エラーメッセージは日本語 + `ERROR_MESSAGES` オブジェクトで定数化
- NFR4: UI 層（`components/`, `app/`）とビジネスロジック（`lib/`）を分離
- NFR5: ESLint・Prettier のチェックがすべて通ること

## 実装のヒント

### バッチ更新条件変更（#1780）

- バッチ処理のメインループを「取引所 → ティッカー」の二重ループに変更
- 現在時刻と取引所の営業時間・タイムゾーンから「登録されるべき最新サマリー日付」を算出するロジックを共通関数として分離
- サマリーの状態判定を「静的解析未実施」と「AI 解析未実施」の2条件に分けて処理
- パターン分析は各パターン単位で未実施かどうかを判定し、追加パターン対応を容易に

### サマータイム考慮（#1779）

- 取引所設定に夏時間フラグまたはタイムゾーン情報（DST 考慮）を持たせる
- 営業時間の判定は UTC オフセットのみでなく DST を加味して計算

### チャートのコンポーネント化（#1781・#1782）

- チャートを再利用可能な React コンポーネントに抽出
- props で表示データ・設定を受け取る設計
- 複数画面（サマリー、詳細、ウォッチリスト）への展開を想定した汎用的なインタフェース

### サマリー改善（#1783・#1784）

- テクニカルパターンの表示領域をサマリーカードに追加
- アラート設定 UI をサマリー画面から操作可能にする
- 表示内容はユーザーが設定でカスタマイズできる構成を検討

### ウォッチリスト脱却（#1785）

- 現状のモック/ハードコーディングを排除し、DynamoDB 等の実データソースと連携
- ウォッチリストの CRUD 操作を API 経由で完結させる設計
