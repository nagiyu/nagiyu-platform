# さくっとクリップ 広告表示機能 - 要件定義

---

## 1. 背景・目的

**背景**:

- さくっとクリップは動画の見どころ抽出に AWS Batch を使用しており、処理中（PENDING/PROCESSING）にユーザーを数十秒〜数分待機させる
- この待ち時間を有効活用し、動画広告を表示することでサービスの広告収入を確保したい

**目的**:

- 処理中画面（SCR-002）に Google IMA SDK による動画広告を組み込む
- 広告再生が完了（またはスキップ）した後のみ、見どころ確認画面へのボタンを有効化する
- 広告の有効・無効を環境変数で切り替えられるようにして、開発・テスト環境での影響を排除する

---

## 2. 追加ユースケース

### UC-004: 動画広告の視聴

- **概要**: ユーザーが処理中画面で動画広告を視聴する
- **アクター**: 動画投稿者
- **前提条件**: UC-001 が完了し、処理中画面（SCR-002）を表示している（ジョブステータスが PENDING または PROCESSING）
- **正常フロー**:
    1. ユーザーが処理中画面を表示する
    2. ジョブステータスが PENDING または PROCESSING になると広告の再生が開始される
    3. 広告が再生完了（またはスキップ可能な広告はスキップ）する
    4. 広告エリアが非表示になる
    5. ジョブステータスが COMPLETED になった後、見どころ確認ボタンが有効化される
- **代替フロー**:
    - 広告が処理完了（COMPLETED）より先に終わった場合: 広告エリアは消え、処理中 UI はそのまま表示され続ける。COMPLETED になり次第ボタンが有効化される
    - 処理が COMPLETED より先に完了した場合: 広告が終わるまでボタンは非表示のまま。広告終了後にボタンが有効化される
- **例外フロー**:
    - 広告ブロッカーにより IMA SDK のロードが失敗した場合: フォールバックとしてボタンを即座に有効化する
    - VAST タグ URL が未設定（空）の場合: 広告をスキップし、処理完了次第ボタンを有効化する（開発・テスト環境向け）
    - 広告の在庫がない（no fill）場合: フォールバックとしてボタンを即座に有効化する
    - 広告読み込みエラーが発生した場合: フォールバックとしてボタンを即座に有効化する

## 3. 追加機能要件

| 機能ID | 機能名 | 説明 | 優先度 |
|--------|--------|------|--------|
| F-011 | 処理中広告表示 | 処理中（PENDING/PROCESSING）に Google IMA SDK で動画広告を表示する | 高 |
| F-012 | 広告完了ゲート | 広告再生完了（または全エラーパスのフォールバック）後かつ処理完了（COMPLETED）後のみ、見どころ確認ボタンを有効化する | 高 |
| F-013 | 広告スキップ制御 | `NEXT_PUBLIC_VAST_TAG_URL` が空または未設定の場合、広告をスキップする（開発・テスト環境向け） | 高 |

---

## 4. 非機能要件への追記

### 4.1 広告に関するエラー耐性

- 広告に関するあらゆるエラー（SDK ロード失敗・no fill・再生エラー）は、**ユーザー体験を損なわないようフォールバック**し、ボタンを有効化しなければならない
- 広告エラーをもってサービス全体の利用を妨げてはならない

### 4.2 環境変数による制御

- `NEXT_PUBLIC_VAST_TAG_URL` が空文字または未定義の場合、広告機能は完全に無効化され、既存の挙動（処理完了次第ボタン表示）を維持する
- CI・E2E テスト環境では当環境変数を設定しないことで広告処理をスキップする

---

## 5. 受け入れ条件

- [ ] PENDING または PROCESSING 状態の処理中画面で動画広告が再生される（`NEXT_PUBLIC_VAST_TAG_URL` 設定時）
- [ ] 広告再生完了後、かつジョブステータスが COMPLETED になった後に「見どころを確認する」ボタンが表示される
- [ ] 広告が先に終わった場合でも、処理完了まではボタンが表示されない
- [ ] 処理が先に COMPLETED になった場合でも、広告終了まではボタンが表示されない
- [ ] FAILED ステータス時は広告を表示せず「再アップロードする」リンクのみ表示する（現行動作の維持）
- [ ] 広告ブロッカー・no fill・エラー時はフォールバックしてボタンが有効化される
- [ ] `NEXT_PUBLIC_VAST_TAG_URL` 未設定時は広告なしで従来通りの動作（COMPLETED でボタン表示）
- [ ] 既存のユニットテスト・E2E テストがすべてパスする
