# さくっとクリップ 広告表示機能 - 外部設計

---

## 1. 変更対象画面

### SCR-002: 処理中画面（変更）

`docs/services/quick-clip/external-design.md` の SCR-002 に対する差分設計。

---

#### 1.1 主要 UI 要素（追加・変更）

| 要素 | 種別 | 説明 | 変更種別 |
|-----|------|------|---------|
| 広告プレイヤー | 動画プレイヤー | 16:9 比率（最大幅 640px）の広告表示エリア。IMA SDK が内部に iframe を挿入する | **追加** |
| 見どころ確認ボタン | ボタン | COMPLETED かつ広告完了後に表示（従来は COMPLETED のみ） | **変更** |

#### 1.2 ユーザーインタラクション（変更）

| 操作 | 結果 | 変更種別 |
|------|------|---------|
| PENDING または PROCESSING 状態で画面が表示される | 広告の再生が自動開始される（`NEXT_PUBLIC_VAST_TAG_URL` 設定時） | **追加** |
| 広告が再生完了またはスキップされる | 広告エリアが非表示になる | **追加** |
| COMPLETED 状態かつ広告完了後に確認ボタンをクリック | SCR-003 へ遷移 | **変更**（条件追加） |

#### 1.3 表示条件・状態（変更）

既存の状態に加えて以下が追加される。

| 状態 | 表示内容 |
|------|---------|
| PENDING / PROCESSING かつ広告未完了 | ステータスチップ + ローディング + **広告プレイヤー** |
| PENDING / PROCESSING かつ広告完了済み | ステータスチップ + ローディング（広告エリアは非表示） |
| COMPLETED かつ広告未完了 | ステータスチップ「処理完了」 + **広告プレイヤー**（ボタンはまだ非表示） |
| COMPLETED かつ広告完了済み | ステータスチップ「処理完了」 + **見どころ確認ボタン**（従来通り） |
| FAILED | ステータスチップ「処理失敗」 + エラーメッセージ + 再アップロードリンク（広告なし・変更なし） |

> `NEXT_PUBLIC_VAST_TAG_URL` が未設定の場合、広告は表示されず「広告完了済み」と同等の状態として扱う（= 従来通りの動作）。

#### 1.4 UI レイアウト（処理中状態）

```
Paper (p: 4)
 └── Stack (spacing: 2)
      ├── Typography "処理中画面"
      ├── [CircularProgress + テキスト]  ← ポーリング中
      ├── Chip (PROCESSING)
      ├── Typography "見どころ抽出を実行中です..."
      │
      ├── [広告プレイヤー]  ← PENDING/PROCESSING かつ !adFinished のみ表示
      │    └── Box (maxWidth: 640px, aspectRatio: 16/9, bgcolor: black)
      │         ├── div (IMA SDK が iframe を挿入)
      │         └── video (広告動画の再生先)
      │
      └── [見どころ確認ボタン]  ← COMPLETED && adFinished のみ表示
```

```
Paper (p: 4)
 └── Stack (spacing: 2)
      ├── Typography "処理中画面"
      ├── Chip (COMPLETED)
      │
      ├── [広告プレイヤー]  ← COMPLETED かつ !adFinished の場合のみ表示
      │    （処理が先に完了した場合、広告がまだ流れていれば継続表示）
      │
      └── [見どころ確認ボタン]  ← adFinished になった時点で表示
```

---

## 2. 画面遷移図（変更なし）

既存の画面遷移図に変更はない。SCR-002 → SCR-003 への遷移条件が「処理完了かつ広告完了」に変わるが、遷移先は同じ。

---

## 3. 設計上の決定事項

| # | 決定 | 理由 |
|---|------|------|
| 1 | 広告は PENDING/PROCESSING 開始時に再生する | 処理の待ち時間を最大限に広告枠として活用するため。COMPLETED 後に流すと待ち時間のないケースで UX が悪化する |
| 2 | 広告完了ゲートは「COMPLETED && adFinished」の AND 条件 | 広告が先に終わっても処理が完了していなければボタンを出さない。処理が先に完了しても広告が終わらなければボタンを出さない |
| 3 | FAILED 時は広告を表示しない | 失敗時にさらに広告を見せるのはユーザー体験を著しく損なうため |
| 4 | エラー・no fill 時はフォールバックで即ボタン有効化 | 広告に起因してサービスが利用不能になることを防ぐため |
| 5 | `NEXT_PUBLIC_VAST_TAG_URL` 未設定で広告無効 | 開発・テスト環境で追加の設定なしに既存の動作を維持できる |
