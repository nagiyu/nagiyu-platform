# 計画書の作成

## 概要

nagiyu-platform のプロジェクト計画書を作成する。
本タスクは、プラットフォーム全体の開発計画・方針をドキュメントとして整備し、今後の開発の基盤とすることを目的とする。

## 関連情報

- タスクタイプ: プラットフォーム

## 要件

### 機能要件

- FR1: プラットフォーム全体の概要・目的を記述した計画書を作成する
- FR2: 各サービス（tools, auth, admin, codec-converter, niconico-mylist-assistant, stock-tracker）の役割と現状を整理する
- FR3: インフラ構成（共通インフラ・サービス別インフラ）の概要を記載する
- FR4: 開発ワークフロー（ブランチ戦略・CI/CD）を計画に反映する
- FR5: 今後の開発優先度・ロードマップを示す

### 非機能要件

- NFR1: ドキュメントは `docs/` 配下の既存ドキュメントと整合性を保つ
- NFR2: Markdown フォーマット（4スペースインデント）に準拠する
- NFR3: 実コードへの参照は最小限にし、ドキュメント単体で理解可能にする

## 実装のヒント

- 計画書は `docs/` 配下の各ドキュメント（README.md、branching.md、development/*.md など）を参照・統合して作成する
- 既存サービスの状況は `services/` および `docs/services/` から読み取る
- インフラ構成は `docs/infra/` から読み取る
- ドキュメント駆動開発の方針（`docs/README.md`）に従い、計画書は将来の実装エージェントへの入力として機能する形式にする

## タスク

### Phase 1: 情報収集

- [ ] T001: 各サービスディレクトリ（`services/`）の現状確認
- [ ] T002: インフラ定義（`infra/`）の現状確認
- [ ] T003: 既存ドキュメント（`docs/`）の整理・棚卸し

### Phase 2: 計画書の作成

- [ ] T004: プラットフォーム概要・目的の記述
- [ ] T005: 各サービスの役割・現状・今後の方針の整理
- [ ] T006: インフラ構成の概要整理
- [ ] T007: 開発ワークフローの記述
- [ ] T008: ロードマップ・優先度の策定

### Phase 3: レビューと整備

- [ ] T009: 既存ドキュメントとの整合性確認
- [ ] T010: `docs/README.md` の目次に計画書へのリンクを追加（必要な場合）
