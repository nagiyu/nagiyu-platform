# 保有株式削除時に売りアラートも連動削除する

## 概要

Stock Tracker にて、保有株式を削除する際に、同一ティッカーの売りアラート（`Mode: 'Sell'`）も合わせて削除する。
削除処理はサイレントに行わず、確認ダイアログで該当する売りアラートの一覧を表示してから実行する。

## 関連情報

- Issue: #（Stock Tracker にて、保有株式の削除に合わせて売りアラートも削除する）
- タスクタイプ: サービスタスク（stock-tracker）

## 要件

### 機能要件

- FR1: 保有株式の削除ダイアログを開いた際、同一ティッカーの売りアラートが存在する場合はその一覧を表示する
- FR2: OK ボタン押下で保有株式の削除と、同一ティッカーの売りアラートの削除を実行する
- FR3: 売りアラートが存在しない場合は、現行と同じ確認ダイアログ（売りアラートの表示なし）を表示する
- FR4: 削除処理はアトミックではないが、エラーが発生した場合はユーザーに通知する

### 非機能要件

- NFR1: ダイアログ表示時に売りアラートを事前取得し、ユーザー体験を損なわないレスポンス速度を維持する
- NFR2: テストカバレッジ 80% 以上を維持する
- NFR3: 既存の削除処理（保有株式のみ削除）の挙動を壊さない

## 実装のヒント

### 現状の削除フロー

```
削除ボタン
  → handleOpenDeleteDialog(holding)
  → deleteDialogOpen = true
  → ダイアログ表示（保有株式情報のみ）
  → 削除ボタン押下
  → handleDelete()
  → DELETE /api/holdings/[id]
  → 成功メッセージ表示
```

### 変更方針

**1. ダイアログ表示前の売りアラート取得**

`handleOpenDeleteDialog` 内でダイアログを開く前に、対象ティッカーの売りアラートを取得して state に格納する。
売りアラート一覧の取得は既存の `GET /api/alerts` を利用し、クライアント側でティッカーIDと Mode でフィルタリングする。

**2. 削除確認ダイアログの更新**

`holdings/page.tsx` の削除確認ダイアログに「売りアラートも削除される」旨の警告と、対象売りアラートの一覧を追加表示する。
保有株式の情報は現行のまま維持する。

**3. 削除処理の更新（holdings/page.tsx）**

`handleDelete` にて、保有株式削除後に売りアラートを順次削除する処理を追加する。
各アラートは `DELETE /api/alerts/[id]` で個別に削除する（既存 API をそのまま利用）。

**4. コア層・API 層への変更**

- `AlertRepository` インターフェースへの新メソッド追加は**不要**（クライアント側フィルタリングで対応可能）
- バックエンド API の変更は**不要**（既存の `GET /api/alerts` と `DELETE /api/alerts/[id]` を再利用）

### 注意点

- `Mode: 'Sell'` のみを対象にする（買いアラートは削除しない）
- 売りアラートが 0 件の場合は通常の削除ダイアログと同じ動作にする
- 売りアラート取得中にローディング状態を表示し、完了後にダイアログを開く
- 削除エラーが発生した場合、エラーメッセージをユーザーに通知する
