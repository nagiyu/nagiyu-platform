# ニコニコマイリストアシスタント バッチ - Playwright Docker イメージ更新 要件定義

---

## 1. ビジネス要件

### 1.1 背景・目的

ニコニコマイリストアシスタントの自動登録バッチにおいて、Playwright の npm パッケージバージョンが `1.59.1` に更新された一方で、バッチの Docker イメージが `mcr.microsoft.com/playwright:v1.58.0-jammy` のままとなっている。

このバージョン不一致により、コンテナ起動時に Playwright が要求するブラウザ実行ファイル（`chrome-headless-shell`）が Docker イメージ内に存在せず、ブラウザの起動に失敗している。本バグを修正し、本番環境でのバッチ正常稼働を回復させることを目的とする。

### 1.2 対象ユーザー

- プライマリーユーザー: ニコニコマイリストアシスタントのエンドユーザー（マイリスト自動登録バッチの実行を依頼するユーザー）

### 1.3 ビジネスゴール

- マイリスト自動登録バッチが本番環境で正常に動作すること
- Playwright のパッケージバージョンと Docker イメージバージョンを常に一致させる運用方針を確立すること

---

## 2. 機能要件

### 2.1 ユースケース

#### UC-001: マイリスト自動登録バッチの正常実行

- **概要**: ユーザーが登録リクエストを送信した際、バッチが Playwright を使用してニコニコ動画のマイリスト登録を正常に実行できる
- **アクター**: エンドユーザー（バッチ起動者）
- **前提条件**: Docker イメージの Playwright バージョンと npm パッケージのバージョンが一致している
- **正常フロー**:
    1. バッチコンテナが起動する
    2. Playwright がブラウザ（chrome-headless-shell）を起動する
    3. ニコニコ動画のマイリスト登録処理が実行される
    4. 登録結果が返却される
- **例外フロー**: Playwright バージョン不一致の場合、起動直後にエラーが発生し全件失敗となる

### 2.2 機能一覧

| 機能ID | 機能名 | 説明 | 優先度 |
| ------ | ------ | ---- | ------ |
| F-001 | Docker イメージバージョン更新 | バッチ Dockerfile の base イメージを `playwright:v1.59.1-jammy` に更新する | 高 |

### 2.3 想定画面の概要

- 画面変更なし（バッチのインフラ・ビルド設定のみの変更）

---

## 3. 非機能要件

### 3.1 パフォーマンス要件

| 項目 | 要件 |
| ---- | ---- |
| 応答時間 | 既存と同等（バッチ処理時間に影響なし） |
| スループット | 既存と同等 |
| 同時接続数 | 既存と同等 |

### 3.2 セキュリティ要件

- Playwright 公式 Docker イメージ（`mcr.microsoft.com/playwright`）を使用すること
- 最新の公式イメージを使用することでセキュリティパッチを適用する

### 3.3 可用性要件

| 項目 | 要件 |
| ---- | ---- |
| 稼働率 | バッチが正常に起動・実行できること |
| RTO (復旧目標時間) | 本 PR のマージ後、次回デプロイで即時回復 |
| RPO (復旧目標時点) | N/A（データ損失なし） |

### 3.4 保守性・拡張性要件

- Playwright のバージョン更新時は、npm パッケージと Docker イメージのバージョンを同時に更新すること
- `package.json` の `playwright` バージョンと Dockerfile の `mcr.microsoft.com/playwright` タグを常に一致させること

---

## 4. ドメインオブジェクト

| エンティティ | 説明 |
| ----------- | ---- |
| Dockerfile | バッチコンテナのビルド定義ファイル。base イメージのバージョン指定を含む |
| Playwright | ブラウザ自動化ライブラリ。npm パッケージと Docker イメージの両方でバージョン管理される |

---

## 5. スコープ外

- ❌ Playwright のさらなるバージョンアップ（1.59.1 以降への更新）
- ❌ バッチの処理ロジックの変更
- ❌ web サービスや core への変更
- ❌ インフラ（CDK/CloudFormation）の変更

---

## 6. 受け入れ条件

- [ ] Dockerfile の base イメージが `mcr.microsoft.com/playwright:v1.59.1-jammy` になっている
- [ ] `package.json` の `playwright` バージョン（`1.59.1`）と Dockerfile のイメージタグ（`v1.59.1`）が一致している
- [ ] Docker イメージをビルドして、ブラウザ起動エラーが発生しないこと
