# 機能仕様書: Stock Tracker AI 解析機能

**機能ブランチ**: `001-stock-ai-analysis`
**作成日**: 2026-03-03
**ステータス**: ドラフト
**入力**: ユーザーの説明: "Stock Tracker のサマリーにて、AI による解析を追加する。バッチにてサマリーを取得した後、直近の値段の移り変わり、パターン分析の結果をもとに、AI による解析を行う。AI で解析する際、値段の移り変わりやパターン分析の根拠となる情報も Web 検索を用いて結果に含められるのが理想。ユーザーは、サマリー画面の詳細ダイアログにて、AI による解析結果を閲覧することができる。AI は OpenAI を用いる。API キーは Secrets Manager に保持する。"

## ユーザーシナリオとテスト *(必須)*

### ユーザーストーリー 1 - サマリー詳細ダイアログで AI 解析結果を閲覧する (優先度: P1)

株式投資家がサマリー画面を開き、特定銘柄の詳細ダイアログを表示すると、その銘柄について AI が生成した解析テキストを確認できる。AI 解析テキストには価格動向の解釈、パターン分析の説明、関連する市場情報が含まれる。

**この優先度の理由**: ユーザーが直接体験する価値の核心。投資判断の補助情報として最も重要。

**独立したテスト**: サマリーレコードに AI 解析テキストが保存されている状態で、詳細ダイアログを開いて表示を確認することで単独テスト可能。

**受け入れシナリオ**:

1. **前提** バッチが正常完了し AI 解析テキストが保存されている、**操作** ユーザーがサマリー画面で特定銘柄の詳細ダイアログを開く、**結果** ダイアログ内の「AI 解析」セクションに解析テキストが表示される
2. **前提** AI 解析テキストが未生成の銘柄がある（バッチ未実行または初回バッチ前）、**操作** ユーザーがその銘柄の詳細ダイアログを開く、**結果** 「AI 解析」セクションに「AI 解析はまだ生成されていません」と表示され、他のサマリー情報は正常表示される
3. **前提** AI 解析テキストの生成が失敗した銘柄がある（OpenAI API エラー等）、**操作** ユーザーがその銘柄の詳細ダイアログを開く、**結果** 「AI 解析」セクションに「AI 解析の取得に失敗しました」と表示され、他のサマリー情報は正常表示される

---

### ユーザーストーリー 2 - バッチ処理で AI 解析が自動生成される (優先度: P1)

バッチ処理実行時、サマリーデータを取得した後、直近の価格推移（OHLC）とパターン分析の結果を入力として OpenAI が解析を実行し、結果をサマリーレコードに保存する。解析時には Web 検索で得た関連市場情報も活用される。

**この優先度の理由**: AI 解析結果をユーザーへ提供するための基盤であり、UI 表示機能と同等の重要度。

**独立したテスト**: バッチ処理を実行し、各銘柄のサマリーレコードに AI 解析テキストが保存されたことをデータストアで確認することで単独テスト可能。

**受け入れシナリオ**:

1. **前提** バッチがサマリー取得を完了している、**操作** バッチの AI 解析処理が実行される、**結果** 各銘柄に対して価格推移・パターン分析を含むプロンプトで OpenAI が呼び出され、返却されたテキストがサマリーレコードに保存される
2. **前提** バッチが AI 解析処理を実行しようとしている、**操作** OpenAI API が Web 検索機能付きで呼び出される、**結果** 解析テキストに価格動向・パターン解釈に加え、Web 検索で取得した関連市場情報が反映される

---

### ユーザーストーリー 3 - サマリー画面の更新ボタンでバッチをキックし AI 解析を取得する (優先度: P2)

株式投資家がサマリー画面の既存の更新ボタンを押すと、バッチが手動でキックされる。バッチ完了後、AI 解析を含む最新のサマリー情報がサマリー画面に反映される。定期実行のバッチを待たずに任意のタイミングで最新の AI 解析を取得したい場合に利用する。

**この優先度の理由**: 更新ボタンは既存の機能であり新規実装は不要。バッチに AI 解析処理が追加されれば、更新ボタン経由のバッチ実行でも自然に AI 解析結果が得られる。

**独立したテスト**: 更新ボタンを押してバッチをキックした後、バッチ完了時にサマリー画面を確認し、AI 解析テキストが表示されることで単独テスト可能。

**受け入れシナリオ**:

1. **前提** ユーザーがサマリー画面を表示している、**操作** ユーザーが更新ボタンを押してバッチをキックする、**結果** バッチが実行されて AI 解析を含む最新のサマリーが生成され、バッチ完了後にサマリー画面の詳細ダイアログで「AI 解析」セクションが閲覧できる

---

### ユーザーストーリー 4 - AI 解析生成に失敗しても既存機能が継続する (優先度: P2)

OpenAI API のエラー、Secrets Manager の取得失敗、ネットワーク障害などが発生した場合でも、バッチの既存サマリー取得処理は正常完了し、サマリー画面の価格情報・パターン分析表示は影響を受けない。

**この優先度の理由**: 既存機能の安定性を保護するためのフォールトトレランス要件。AI 障害が他機能に波及しないことが重要。

**独立したテスト**: OpenAI API をエラー応答させた状態でバッチを実行し、既存サマリーデータが正常に保存されることと、バッチが正常終了することを確認することで単独テスト可能。

**受け入れシナリオ**:

1. **前提** OpenAI API が利用不可またはエラーを返す状態、**操作** バッチが実行される、**結果** AI 解析処理はスキップ（またはエラーログ記録）されるが、他のバッチ処理は正常完了し既存サマリーデータが保存される
2. **前提** Secrets Manager から API キーが取得できない状態、**操作** バッチが実行される、**結果** AI 解析処理は安全にスキップされ、バッチ全体の処理は継続する

---

### エッジケース

- API キーが Secrets Manager に存在しない場合はどうなるか？ → AI 解析処理をスキップし、エラーをログに記録する
- OpenAI のレート制限に達した場合はどうなるか？ → 指数バックオフで最大 3 回リトライし、それでも解消しない場合は当該銘柄の AI 解析をスキップし、他銘柄の処理は継続する
- 解析対象銘柄数が多くバッチタイムアウトに近づいた場合はどうなるか？ → 完了できた分の解析結果を保存し、残りはスキップする
- Web 検索が失敗した場合はどうなるか？ → Web 検索なしで価格推移・パターン分析のみを基に解析を生成する
- 更新ボタンを押した際に AI 解析生成が進行中の場合はどうなるか？ → バッチは既存の排他制御に従う（既存動作に準ずる）

## 要件 *(必須)*

### 機能要件

- **FR-001**: バッチ処理はサマリーデータ取得完了後、各銘柄の直近価格推移（OHLC）とパターン分析結果を OpenAI へのプロンプトに含めて AI 解析を実行することを MUST とする
- **FR-002**: バッチ処理は OpenAI API キーを AWS Secrets Manager から取得することを MUST とする
- **FR-003**: バッチ処理は AI 解析時に価格動向・パターン分析の根拠となる情報を Web 検索で取得し解析結果に含めることを SHOULD とする。実装は OpenAI Responses API（モデル: `gpt-4o`）の `web_search_preview` ツールを使用する
- **FR-004**: バッチ処理は AI 解析結果テキストを日本語でサマリーレコードに保存することを MUST とする
- **FR-005**: AI 解析処理（OpenAI API 呼び出し、Secrets Manager 取得、Web 検索）が失敗した場合、バッチ処理全体を停止させずにスキップすることを MUST とする
- **FR-006**: AI 解析処理の失敗はログに記録することを MUST とする
- **FR-010**: バッチ処理は OpenAI API 呼び出し失敗時（ネットワークエラー・タイムアウト・一時的なレート制限超過）に対して指数バックオフで最大 3 回リトライし、それでも失敗した場合のみ当該銘柄の AI 解析をスキップすることを MUST とする
- **FR-007**: サマリー取得 API は AI 解析テキストをレスポンスに含めることを MUST とする
- **FR-008**: サマリー詳細ダイアログは「AI 解析」セクションを表示し、AI 解析テキストを閲覧できることを MUST とする
- **FR-009**: AI 解析テキストが存在しない場合、ダイアログはその状態に応じたメッセージを表示し、他のサマリー情報を正常表示し続けることを MUST とする。具体的には、未生成（バッチ未実行等）の場合は「AI 解析はまだ生成されていません」、生成失敗の場合は「AI 解析の取得に失敗しました」と表示する

### 主要エンティティ *(データを扱う機能の場合)*

- **サマリーレコード**: 銘柄ごとのサマリー情報を表す。既存の価格情報・パターン分析に加え、AI 解析テキスト（任意）を属性として持つ
- **AI 解析テキスト**: 価格推移・パターン分析・Web 検索結果を基に OpenAI（`gpt-4o`）が生成した日本語の解析文。サマリーレコードに紐付く。最大 2000 文字。状態として「未生成」「生成済み」「生成失敗」の 3 値を区別できるよう設計する
- **OpenAI API キー**: Secrets Manager に保管される認証情報。バッチ処理実行時に取得して使用する

## 成功基準 *(必須)*

### 測定可能な成果

- **SC-001**: 全対象銘柄の AI 解析がバッチ処理の制限時間内（既存バッチのタイムアウト以内）に完了する
- **SC-002**: OpenAI API やネットワーク障害が発生した場合でも、既存の価格情報・パターン分析の表示が 100% 正常に動作し続ける
- **SC-003**: AI 解析テキストを保持するサマリーレコードに対して、詳細ダイアログが即時（追加のネットワークリクエストなし）に解析内容を表示できる
- **SC-004**: 生成された AI 解析テキストに価格動向の解釈・パターンの説明・関連市場情報（Web 検索結果）が含まれている
- **SC-005**: Secrets Manager からの API キー取得が失敗した場合、AI 解析処理が安全にスキップされ、バッチの他処理が正常完了する

## 制約と前提条件

### 制約

- AI エンジンは OpenAI に限定する（他の AI サービスは対象外）
- API キーの管理は AWS Secrets Manager に限定する（ハードコードや環境変数への直接設定は禁止）
- AI 解析はバッチ処理実行時に生成される（定期実行またはサマリー画面の既存の更新ボタンによる手動キックを含む）

### 前提条件

- 既存バッチ処理でサマリーデータ（価格推移・パターン分析）が取得・保存されている
- バッチ 1 回あたりの解析対象銘柄数は 100 銘柄以下を想定する（超過する場合は設計見直しを要検討）
- AWS Secrets Manager へのアクセス権限がバッチの実行ロールに付与される
- OpenAI Responses API（モデル: `gpt-4o`）および `web_search_preview` ツールが利用可能な状態にある

### 対象外

- OpenAI 以外の AI サービスの利用
- AI 解析結果の編集・フィードバック機能
- 複数銘柄の AI 解析を比較する機能
