# 詳細表示トグル機能 - テストデータ

## テスト用の乗り換え案内テキスト

以下のテキストをコピーして、乗り換え変換ツールで試してください。

```
東京駅 ⇒ 大阪駅
2025年1月15日(水)
08:00 ⇒ 10:30
------------------------------
所要時間 2時間30分
運賃[IC優先] 13,320円
乗換 0回
距離 515.4 km
------------------------------

■東京駅
↓ 08:00〜10:30
↓ 東海道新幹線のぞみ 新大阪行
↓ 16番線発 → 23番線着
■大阪駅
```

## 期待される動作

### デフォルト設定（すべて有効、距離・番線情報のみ無効）

```
【乗り換え案内】2025年1月15日(水)
東京駅 08:00 → 大阪駅 10:30
所要時間: 2時間30分 / 運賃: 13,320円 / 乗換: 0回

[ルート]
東京駅 (08:00〜10:30)
→ 東海道新幹線のぞみ 新大阪行
大阪駅
```

### すべて有効（距離・番線情報も表示）

```
【乗り換え案内】2025年1月15日(水)
東京駅 08:00 → 大阪駅 10:30
所要時間: 2時間30分 / 運賃: 13,320円 / 乗換: 0回 / 距離: 515.4 km

[ルート]
東京駅 (08:00〜10:30)
→ 東海道新幹線のぞみ 新大阪行 [16番線発 → 23番線着]
大阪駅
```

### 最小構成（日付、出発地・到着地、時刻のみ）

```
【乗り換え案内】2025年1月15日(水)
東京駅 08:00 → 大阪駅 10:30
```

### ルート詳細なし

```
【乗り換え案内】2025年1月15日(水)
東京駅 08:00 → 大阪駅 10:30
所要時間: 2時間30分 / 運賃: 13,320円 / 乗換: 0回
```

## 検証項目

- [ ] 表示設定のAccordionが展開/折りたたみできる
- [ ] 各チェックボックスをクリックすると即座に出力が更新される
- [ ] ルート詳細の親チェックボックスを外すと子項目も無効になる
- [ ] 子項目をチェックすると親項目も自動的に有効になる
- [ ] 設定を変更してページをリロードしても設定が保持される
- [ ] プライベートモードでもエラーなく動作する（設定は保存されない）
- [ ] 距離と番線情報はデフォルトで非表示
- [ ] その他の項目はデフォルトで表示

## LocalStorage確認方法

ブラウザの開発者ツール (F12) → Application → Local Storage → 該当ドメイン

キー: `transit-converter-display-settings`

値の例:
```json
{
  "showDate": true,
  "showDepartureArrival": true,
  "showTime": true,
  "showDuration": true,
  "showFare": true,
  "showTransferCount": true,
  "showDistance": false,
  "showRouteDetails": true,
  "showTimeRange": true,
  "showLineName": true,
  "showPlatform": false
}
```
