## Quick Clip の使い方ガイド

このページでは、Quick Clip を使って動画のハイライト抽出・クリップ切り出しを行う手順を解説します。

## 事前準備

- 対応ブラウザ（Chrome・Edge 最新版）をご用意ください。
- 処理したい動画ファイル（MP4 形式）を手元に準備してください。
- アカウント登録・ログインは不要です。ブラウザからそのまま利用できます。

---

## クリップ切り出しの手順

### ステップ 1: サービスへのアクセス

[https://quick-clip.nagiyu.com](https://quick-clip.nagiyu.com) にアクセスします。アカウント登録・ログインは不要で、そのまま動画をアップロードできます。

### ステップ 2: 動画のアップロード

1. ファイル選択ダイアログから動画ファイルを選択するか、ドラッグ＆ドロップでアップロードエリアに動画をドロップします。
2. 対応フォーマット: MP4 のみ
3. 最大ファイルサイズ: 20 GB（大容量動画も対応）

### ステップ 3: 解析処理の完了待ち

アップロードが完了すると自動的に AWS Batch のジョブが開始されます。処理状況はページ上のステッパー（アップロード → 解析 → 切り出し）でリアルタイムに確認できます。処理時間の目安は 10 分の動画で 2〜5 分程度です。

### ステップ 4: 見どころの確認と採否チェック

処理が完了すると「見どころを確認する」ボタンが有効になります。ボタンをクリックすると、抽出された見どころ一覧が表示されます。各行をクリックすると左パネルでプレビューを確認でき、「使える」「使えない」「未確認」から採否を選択できます。

### ステップ 5: 時間調整（任意）

左パネルで開始・終了時刻（秒単位）を直接入力して調整できます。フォーカスを外すとクリップが自動再生成されます。

### ステップ 6: ZIP ダウンロード

「使える」にした見どころのクリップをまとめて ZIP でダウンロードできます。「ZIP ダウンロード」ボタンをクリックしてください。生成されたファイルは 24 時間保存されます。期限内にダウンロードしてください。

## 処理状況の確認

処理中ページでジョブのステータスを確認できます。

| ステータス | 説明                   |
| ---------- | ---------------------- |
| 処理待ち   | ジョブキューに登録済み |
| 処理中     | AWS Batch で実行中     |
| 処理完了   | ダウンロード可能       |
| 処理失敗   | エラー発生             |

## 使ってみて気づいたこと（一人称メモ）

自分で使ってみて最初に戸惑ったのが「アップロード中はタブを閉じないでほしい」という制約です。アップロードが終わった後の解析処理中はタブを閉じても問題なく続きますが、アップロード自体が途中で切れると再チャレンジが必要になります。大容量ファイルを扱うときはアップロード完了を確認してからタブを離れるようにすると安心です。また、データの有効期限は処理開始から 24 時間です。夜中に処理を走らせて翌日ダウンロードしようとしたら消えていた、という経験を何度かしたので、処理完了後はなるべくすぐにダウンロードすることをおすすめします。
