## nagiyu Admin の使い方ガイド

このページでは、nagiyu Admin の各管理機能の使い方を解説します。

## アクセス方法

[https://admin.nagiyu.com](https://admin.nagiyu.com) にアクセスし、管理者権限を持つ nagiyu アカウントでログインします。管理者権限のないアカウントではアクセスできません。

---

## ダッシュボード

ログイン後のダッシュボードでは、プラットフォーム全体の概況を一目で確認できます。

- **アクティブユーザー数**: 過去 24 時間・7 日間・30 日間のアクティブユーザー数
- **サービス稼働状況**: 各サービスの現在の状態（正常・警告・停止）
- **直近のエラー**: 直近 1 時間に発生したエラーの件数と内容
- **処理ジョブ状況**: AWS Batch のジョブキュー状況（待機中・処理中・完了・失敗）

## ユーザー管理

### ユーザー一覧の確認

左サイドバーの「ユーザー管理」をクリックします。全ユーザーの一覧が表示されます。検索ボックスでメールアドレス・表示名・Google アカウント ID で絞り込みができます。

### ユーザーの詳細確認

ユーザー一覧でユーザーをクリックすると詳細ページが表示されます。

- 登録日時・最終ログイン日時
- 利用しているサービスと最終利用日
- 各サービスでのデータ使用量
- 過去のログイン履歴

### 管理者権限の付与・剥奪

1. ユーザー詳細ページの「権限」タブを開きます。
2. 「管理者権限を付与する」ボタンをクリックします。
3. 確認ダイアログで「付与する」をクリックします。

権限剥奪も同様の手順で行います。

### アカウントの一時停止

不正利用が疑われるユーザーのアカウントを一時停止します。

1. ユーザー詳細ページの「アクション」タブを開きます。
2. 「アカウントを一時停止する」ボタンをクリックします。
3. 停止理由を入力して「実行する」をクリックします。

停止中はユーザーがログインしようとすると停止中のメッセージが表示されます。

## サービス設定

### 機能フラグの管理

1. 左サイドバーの「サービス設定」→対象サービス名をクリックします。
2. 「機能フラグ」タブを開きます。
3. 変更したい機能のトグルスイッチを切り替えます。
4. 変更は即座に反映されます。

### メンテナンスモードの設定

1. サービス設定の「メンテナンス」タブを開きます。
2. 「メンテナンスモードを有効にする」トグルをオンにします。
3. メンテナンス予定時間と案内文を入力します。
4. 「保存する」をクリックして反映します。

---

## システム監視

### エラーログの確認

1. 左サイドバーの「監視」→「エラーログ」をクリックします。
2. サービス・日時・ログレベルでフィルタリングできます。
3. 各ログをクリックするとスタックトレースの詳細を確認できます。

### アラートの設定

1. 「監視」→「アラート設定」をクリックします。
2. 「新しいアラートを追加」をクリックします。
3. 監視対象・条件・通知先（メール・Slack Webhook URL）を設定します。
4. 「保存する」をクリックして完了します。
