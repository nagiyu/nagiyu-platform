## nagiyu Admin とは

nagiyu Admin は、nagiyu プラットフォーム全体を管理するための管理画面サービスです。[https://admin.nagiyu.com](https://admin.nagiyu.com) からアクセスできます（管理者権限が必要です）。Auth サービスとの SSO 連携により、`admin` ロールを持つアカウントのみが利用できます。

## 主な機能

### ダッシュボード

ログイン後のダッシュボードでユーザー情報（名前・メールアドレス・保有ロール）と JWT の有効期限を確認できます。`admin` ロール保持者にはプッシュ通知の購読ボタンが表示されます。

### エラーイベント閲覧

CloudWatch Alarm 由来のエラーイベントを時系列で一覧・詳細表示します。発生サービス・時刻・内容をブラウザから直接確認できます。エラーイベントには TTL（有効期限）が設定されており、期限経過後は自動削除されます。

### Web Push 通知

CloudWatch Alarm が発火した際に管理者端末へプッシュ通知を配信します。通知をタップすると対応するエラーの詳細ページに直接遷移できます。スマートフォンで PWA としてホーム画面に追加すると、アプリと同様の感覚で通知を受け取れます。

## アクセス権限

nagiyu Admin へのアクセスには管理者権限が必要です。`admin` ロールのない nagiyu アカウントでアクセスしようとすると、403 エラーが表示されアクセスが拒否されます。

## セキュリティ

管理画面へのアクセスはすべて nagiyu Auth による JWT 検証を経由します。SNS からの通知は AWS 公開鍵で署名を検証し、偽造通知を排除します。Web Push は VAPID 認証を使用してサーバーの正当性を証明します。

## 対象ユーザー

nagiyu Admin は、nagiyu プラットフォームの管理者・運営者を対象としています。一般ユーザーの利用は想定していません。

## 背景・開発動機

nagiyu プラットフォームが複数のサービスを抱えるようになると、各サービスの状態を個別の AWS コンソール画面を行き来しながら確認する運用上の手間が課題でした。CloudWatch のアラームが鳴っても、どのサービスに何が起きたのかをすぐ把握できない状況が続いていました。

出発点は Auth サービスとの SSO 連携確認という小さなスコープでしたが、実運用の中でエラー検知から管理者端末へのプッシュ通知（Web Push）まで通知経路を一本化するアーキテクチャに育ちました。CloudWatch Alarm が発火したらすぐ手元の端末に届き、通知をタップするだけでエラー詳細ページを直接開けることが、私が求めていた運用体験です。

## 想定ユーザー

nagiyu Admin は一般公開を前提としない内部管理ツールです。私自身を含む運営チームメンバーが主な利用者で、`admin` ロールが付与されたアカウントからのみアクセスできます。

典型的な使い方は次の通りです。Admin を PWA としてホーム画面に追加しておくと、サービスに異常が発生した瞬間にプッシュ通知が届きます。通知をタップすると管理画面のエラー詳細ページに直接遷移し、発生サービス・時刻・内容をその場で確認できます。

一般ユーザーが各サービスを利用したい場合は nagiyu Admin の対象外です。エンドユーザー向けの機能は各サービスのアプリケーション側に実装されています。
