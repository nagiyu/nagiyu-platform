## よくある質問

### Q. どのファイルフォーマットに対応していますか？

**A.** 入力は MP4 のみに対応しています。出力は H.264（.mp4）・VP9（.webm）・AV1（.webm）の動画コーデックに対応しています。音声のみの変換には対応していません。

### Q. AV1 と H.264 はどちらを選べばよいですか？

**A.** 互換性を重視する場合は H.264、圧縮効率を重視する場合は AV1 を推奨します。AV1 は H.264 比で同等画質でファイルサイズを約 50% 削減できますが、エンコード時間が長くなります。古いデバイスでの再生互換性を考慮する場合は H.264 が安全です。

### Q. 変換にかかる時間はどのくらいですか？

**A.** 変換時間はファイルサイズ・解像度・選択したコーデックによって異なります。目安として、1 時間の 1080p 動画を H.264 に変換する場合で約 10〜20 分です。AV1 などエンコード負荷が高いコーデックでは、さらに時間がかかる場合があります。

### Q. 変換中にブラウザを閉じても問題ありませんか？

**A.** 問題ありません。変換処理は AWS Batch 上で実行されており、ブラウザを閉じても処理は継続されます。後でダッシュボードにアクセスして処理結果を確認・ダウンロードできます。

### Q. 変換の品質を細かく設定できますか？

**A.** 現時点では品質パラメータのカスタム設定はできません。CRF 値はコーデックごとに固定されており（H.264: 23 / VP9: 30 / AV1: 30）、解像度・ビットレート・フレームレートの指定 UI も現在は提供していません。出力コーデック（H.264・VP9・AV1）のみ選択できます。

### Q. 変換できるファイルの最大サイズは？

**A.** 最大 500MB のファイルに対応しています。AWS Batch による処理のため、ブラウザのメモリ制限に影響されません。

### Q. 変換済みファイルはいつまで保存されますか？

**A.** 変換完了から 24 時間保存されます。必要なファイルは期限内にダウンロードしてください。

## 作者からのよくある補足

このサービスを作って自分でも使う中で、よく聞かれそうだと感じた点を補足します。

### Q. なぜ入力が MP4 だけなのですか？

**A.** 私が MVP の範囲を決める際に、まずは最も一般的なフォーマットである MP4 に絞って確実に動くものを届けることを優先しました。AVI・MOV・MKV などの入力対応は今後の拡張として考えていますが、現時点では MP4 のみです。他の形式のファイルは、事前に別のツールで MP4 に変換してからご利用ください。

### Q. 処理中にページを離れてもジョブ ID はどこで確認できますか？

**A.** ジョブ ID はファイルアップロード完了後に URL に含まれます。たとえば `https://codec-converter.nagiyu.com/jobs/{ジョブID}` の形式です。私自身も変換中に誤ってタブを閉じてしまったことがありますが、ジョブ ID を URL 履歴から見つけることで無事ダウンロードできました。変換開始後はブックマークするかジョブ ID をメモしておくことをおすすめします。

### Q. 変換が失敗するのはどんなケースですか？

**A.** 主なケースは「MP4 コンテナだが内部のコーデックが FFmpeg で処理できない形式」「変換処理が 2 時間のタイムアウトを超えた」の 2 パターンです。私の経験では、一般的な MP4 ファイルであれば問題なく処理できています。失敗した場合はステータスが FAILED になるので、「再試行」ボタンで再実行してみてください。
